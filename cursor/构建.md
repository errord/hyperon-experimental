## 完整编译步骤指南

以下是在你的 Windows 系统上成功编译和运行 `metta-repl` 的完整步骤：

### 前置条件

你的系统已有：
- **Rust stable** (rustc 1.94.0, x86_64-pc-windows-msvc)
- **Visual Studio 2022 Community** (含 MSVC v14.40)

### 步骤 1：安装 protoc（Protobuf 编译器）

`das` 功能需要 protoc。由于 Chocolatey 需要管理员权限，直接从 GitHub 下载：

```powershell
# 下载 protoc 并解压到用户目录
$url = "https://github.com/protocolbuffers/protobuf/releases/download/v29.3/protoc-29.3-win64.zip"
$outDir = "$env:USERPROFILE\protoc"
$zipFile = "$env:TEMP\protoc.zip"
Invoke-WebRequest -Uri $url -OutFile $zipFile
Expand-Archive -Path $zipFile -DestinationPath $outDir -Force
Remove-Item $zipFile

# 验证安装
& "$outDir\bin\protoc.exe" --version
```

### 步骤 2：编译并运行

由于 `link.exe` 未在默认 PATH 中，需通过 `vcvarsall.bat` 初始化 MSVC 环境。在 **cmd.exe** 中执行：

```cmd
:: 初始化 MSVC x64 编译环境
call "C:\Program Files\Microsoft Visual Studio\2022\Community\VC\Auxiliary\Build\vcvarsall.bat" x64

:: 设置 protoc 路径
set PROTOC=C:\Users\admin\protoc\bin\protoc.exe

:: 进入项目目录
cd /d d:\dev\hyperon-experimental

:: 编译并运行
cargo run --bin metta-repl
```

### 或者：一行命令（在 PowerShell/cmd 中）

```cmd
cmd /c "call ""C:\Program Files\Microsoft Visual Studio\2022\Community\VC\Auxiliary\Build\vcvarsall.bat"" x64 >nul 2>&1 && set PROTOC=C:\Users\admin\protoc\bin\protoc.exe && cd /d d:\dev\hyperon-experimental && cargo run --bin metta-repl"
```

### 可选：永久设置环境变量

为了避免每次都要调用 `vcvarsall.bat`，可以：

1. **PROTOC 环境变量**：在"系统属性 > 环境变量"中添加用户变量 `PROTOC`，值为 `C:\Users\admin\protoc\bin\protoc.exe`

2. **MSVC 环境**：推荐使用 **"x64 Native Tools Command Prompt for VS 2022"**（从开始菜单搜索），它会自动设置好 MSVC 环境，然后只需在其中运行：

```cmd
set PROTOC=C:\Users\admin\protoc\bin\protoc.exe
cd /d d:\dev\hyperon-experimental
cargo run --bin metta-repl
```

### 关键问题总结

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| `link.exe` not found | MSVC 链接器不在 PATH 中 | 使用 `vcvarsall.bat x64` 初始化环境 |
| `protoc` not found | DAS 功能需要 Protobuf 编译器 | 从 GitHub 下载 protoc 并设置 `PROTOC` 环境变量 |

## 完整编译安装步骤（Python 扩展开发模式）

以下是你可以在终端中自己复现的完整步骤。

### 前置条件

你的系统已有：
- Python 3.12.3（`D:\software\python`）
- Rust stable 1.94.0 (x86_64-pc-windows-msvc)
- Visual Studio 2022 Community (MSVC 14.40)
- CMake 3.28.3
- protoc（已安装到 `C:\Users\admin\protoc`）

### 步骤 1：安装工具依赖

```powershell
# 安装 Conan 包管理器
pip install conan==2.19.1

# 创建 Conan 默认 profile
conan profile detect --force

# 安装 cbindgen（C 头文件生成器）—— 需要在 MSVC 环境中
cmd /c "call ""C:\Program Files\Microsoft Visual Studio\2022\Community\VC\Auxiliary\Build\vcvarsall.bat"" x64 >nul 2>&1 && cargo install --force cbindgen"
```

### 步骤 2：修复 Conan Profile

编辑 `C:\Users\admin\.conan2\profiles\default`，确保内容如下：

```ini
[settings]
arch=x86_64
build_type=Release
compiler=msvc
compiler.version=194
compiler.runtime=dynamic
os=Windows
```

### 步骤 3：CMake 配置 + 构建

打开 **cmd.exe**（不是 PowerShell），执行：

```cmd
:: 初始化 MSVC x64 环境
call "C:\Program Files\Microsoft Visual Studio\2022\Community\VC\Auxiliary\Build\vcvarsall.bat" x64

:: 设置 protoc 路径
set PROTOC=C:\Users\admin\protoc\bin\protoc.exe

:: 进入项目目录
cd /d d:\dev\hyperon-experimental

:: 清理并创建 build 目录
rmdir /s /q build
mkdir build
cd build

:: CMake 配置（使用 NMake 单配置生成器）
cmake .. -G "NMake Makefiles" -DCMAKE_BUILD_TYPE=Debug

:: CMake 构建（约 15-20 分钟，首次需编译 OpenSSL 等依赖）
cmake --build .
```

> **注意**: 首次构建需要从源码编译 OpenSSL、zlib、libcheck 等 Conan 依赖，耗时较长。后续构建会使用缓存，速度快很多。

### 步骤 4：开发模式安装 Python 包

```cmd
cd /d d:\dev\hyperon-experimental
pip install -e "./python[dev]"
```

### 步骤 5：验证

```python
python -c "import hyperon; print(hyperon.__version__); from hyperon import MeTTa; m = MeTTa(); print(m.run('!(+ 1 2)'))"
```

输出：
```
0.2.10
[[3]]
```

### 开发模式说明

- **Python 代码修改**（`python/hyperon/` 下的 `.py` 文件）：修改后立即生效，无需重新安装。
- **C++/Rust 代码修改**（`python/hyperonpy.cpp`、`lib/`、`c/` 等）：需要重新执行步骤 3 的 `cmake --build .` 重新编译本地库，Python 层会自动加载新的 `.pyd`。
- `pip show hyperon` 显示 `Editable project location: D:\dev\hyperon-experimental\python` 确认开发模式。

#### 方法：双调试器附加

1. **先启动 Python 调试**：选择 **"混合调试: Python 启动 + Attach Rust"**，它会在入口处暂停

### 四、执行流程总览

一条 MeTTa 代码 `!(+ 1 2)` 从输入到输出的完整路径：

```
[Python 层]
  MeTTa.run("!(+ 1 2)")           # python/hyperon/runner.py
    → hp.metta_run(...)            # hyperonpy C 绑定调用
    
[C FFI 层]
  metta_run()                      # c/src/metta.rs
    
[Rust 核心层]
  Metta::run(parser)               # lib/src/metta/runner/mod.rs
    → RunnerState::run_to_completion()
      → 循环调用 RunnerState::step()
        → interpret(space, expr)   # lib/src/metta/runner/stdlib/mod.rs
          → interpret_init()       # lib/src/metta/interpreter.rs
          → 循环: interpret_step() # 解释器单步执行（核心！）
            → eval / chain / unify 等操作
            → 匹配 + 运算符，调用 GroundedAtom::execute()
              → 执行 1 + 2 = 3
          → state.into_result()    # 返回 [3]

[返回 Python]
  结果包装为 Python Atom 对象 → [[3]]
```

### 六、实用技巧

1. **条件断点**：右键断点 → "编辑断点" → 输入条件表达式（如 Rust 中 `expr.to_string() == "(+ 1 2)"`）
2. **Logpoint**：不暂停但打印变量值，右键行号 → "添加 Logpoint"
3. **Watch 表达式**：在调试面板的 Watch 区域添加 `expr.to_string()` 观察 Atom 的文本表示
4. **Call Stack**：调试暂停时查看调用栈，可以完整看到从入口到当前位置的调用链
